# BreadixAI - Инструкция по запуску

## Что такое BreadixAI?

BreadixAI — это AI-чат с доступом к интернету и разным моделям. Проект состоит из трёх частей:

1. **Сайт** — интерфейс чата, размещён на GitHub Pages (бесплатный хостинг от GitHub)
2. **Cloudflare Worker** — API-прокси, который связывает сайт с AI моделями
3. **OmniRoute** — локальный сервер на твоём компьютере, который предоставляет доступ к AI моделям

### Как это работает?

```
Сайт (GitHub Pages) 
    ↓
Cloudflare Worker (API прокси)
    ↓
Cloudflare Tunnel (временный публичный URL)
    ↓
OmniRoute (локальный сервер на твоём ПК)
    ↓
AI модели (Claude, GPT, и др.)
```

---

## Что нужно запускать каждый раз?

Каждый раз когда хочешь использовать BreadixAI, нужно:

1. **Запустить OmniRoute** — локальный сервер с AI моделями
2. **Запустить start.ps1** — скрипт который создаёт туннель к OmniRoute
3. **Обновить URL в Cloudflare Worker** — чтобы Worker знал новый адрес туннеля

## Пошаговый запуск

### Шаг 1: Запусти OmniRoute

OmniRoute — это программа которая должна быть запущена на твоём компьютере.

**Где найти:**
- Обычно это файл `OmniRoute.exe` или ярлык на рабочем столе
- Или запускается через командную строку

**Как запустить:**
1. Найди и запусти OmniRoute
2. Дождись пока в консоли появится сообщение типа:
   ```
   Server running on http://localhost:20128
   ```
3. **Не закрывай это окно!** OmniRoute должен работать всё время пока ты используешь BreadixAI

**Проверка:** Открой в браузере `http://localhost:20128` — должна открыться страница OmniRoute

---

### Шаг 2: Запусти start.ps1

Этот скрипт создаёт публичный туннель к твоему локальному OmniRoute через Cloudflare.

**Способ 1: Через правую кнопку мыши (самый простой)**
1. Найди файл `start.ps1` в папке проекта `C:\Claude-code\project\breadixwebsite\`
2. Нажми правой кнопкой мыши на файл
3. Выбери **"Run with PowerShell"**
4. Дождись появления информации с Tunnel URL

**Способ 2: Через терминал**
1. Открой PowerShell (Win + X → Windows PowerShell)
2. Перейди в папку проекта:
   ```powershell
   cd C:\Claude-code\project\breadixwebsite
   ```
3. Запусти скрипт:
   ```powershell
   powershell -ExecutionPolicy Bypass -File start.ps1
   ```

**Что должно произойти:**
- Скрипт запустит Cloudflare Tunnel
- Через 5-10 секунд появится новый URL вида:
  ```
  https://abc-def-ghi-jkl.trycloudflare.com
  ```
- Скрипт покажет всю информацию и инструкцию

**Важно:** Не закрывай окно PowerShell! Туннель работает только пока окно открыто.

---

### Шаг 3: Скопируй Tunnel URL

После запуска `start.ps1` ты увидишь примерно такой вывод:

```
================================================================
                         INFORMATION
================================================================

Tunnel URL:
  https://abc-def-ghi-jkl.trycloudflare.com

Website:
  https://x4m9k2p7v3-boop.github.io/BreadixAI/

Worker URL:
  https://breadixai-api.v-e-kikkas.workers.dev

================================================================

IMPORTANT: Update OMNIROUTE_BASE_URL in Cloudflare Worker!
```

**Скопируй Tunnel URL** (строку вида `https://...trycloudflare.com`)

---

### Шаг 4: Обнови OMNIROUTE_BASE_URL в Cloudflare Worker

Теперь нужно сказать Cloudflare Worker новый адрес туннеля.

**Пошагово:**

1. **Открой Cloudflare Dashboard**
   - Перейди по ссылке: https://dash.cloudflare.com/
   - Войди в свой аккаунт

2. **Найди Worker**
   - В левом меню нажми **"Workers & Pages"**
   - Найди в списке **"breadixai-api"** и нажми на него

3. **Открой настройки**
   - Вверху нажми вкладку **"Settings"**
   - Прокрути вниз до раздела **"Variables and Secrets"**

4. **Найди переменную OMNIROUTE_BASE_URL**
   - В списке переменных найди **"OMNIROUTE_BASE_URL"**
   - Справа от неё нажми кнопку **"Edit"** (иконка карандаша)

5. **Вставь новый URL**
   - Удали старое значение
   - Вставь скопированный Tunnel URL (например: `https://abc-def-ghi-jkl.trycloudflare.com`)
   - **Важно:** URL должен быть БЕЗ `/v1` или других путей в конце

6. **Сохрани изменения**
   - Нажми **"Save"**
   - Затем нажми **"Save and Deploy"** (синяя кнопка справа вверху)

7. **Дождись деплоя**
   - Cloudflare обновит Worker (занимает 5-10 секунд)
   - Появится уведомление "Deployment successful"

Теперь всё готово!

**Открой сайт:**
https://x4m9k2p7v3-boop.github.io/BreadixAI/

2. Если AI отвечает — всё работает отлично!

## Частые проблемы и решения

### Проблема: Сайт не отвечает / "Failed to fetch"

**Причины:**
- OmniRoute не запущен
- Туннель не работает
- Неправильный URL в Cloudflare Worker

**Решение:**
1. Проверь что OmniRoute запущен и работает (`http://localhost:20128` открывается)
2. Проверь что окно PowerShell со скриптом `start.ps1` не закрыто
3. Проверь что ты правильно скопировал Tunnel URL в Cloudflare Worker
4. Попробуй перезапустить `start.ps1` и обновить URL заново

---

### Проблема: Туннель не запускается / "Failed to get tunnel URL"

**Причины:**
- `cloudflared.exe` не найден
- Интернет не работает
- Cloudflare временно недоступен

**Решение:**
1. Проверь что файл `C:\Users\user\Downloads\cloudflared.exe` существует
2. Проверь интернет-соединение
3. Попробуй запустить `start.ps1` ещё раз через 1-2 минуты

---

### Проблема: API ошибка / "API key not configured"

**Причины:**
- Не настроены переменные в Cloudflare Worker
- Неправильный API ключ

**Решение:**
1. Открой Cloudflare Dashboard → Workers & Pages → breadixai-api → Settings → Variables
2. Проверь что есть переменные:
   - `OMNIROUTE_KEY` — API ключ для OmniRoute
   - `TAVILY_KEY` — API ключ для поиска
   - `OMNIROUTE_BASE_URL` — URL туннеля
3. Если какой-то переменной нет — добавь её

---

### Проблема: Worker показывает старый URL

**Причина:**
- Cloudflare кеширует старые настройки

**Решение:**
1. Открой Cloudflare Dashboard → Workers & Pages → breadixai-api
2. Нажми **"Save and Deploy"** ещё раз
3. Подожди 30 секунд и попробуй снова

---

### Проблема: PowerShell не запускает скрипт / "Execution Policy"

**Причина:**
- Windows блокирует выполнение PowerShell скриптов

**Решение:**
Запусти скрипт через команду:
```powershell
powershell -ExecutionPolicy Bypass -File start.ps1
```

Или открой PowerShell от имени администратора и выполни:
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
